# PRODUCT

## Target user
People who read books, articles, papers, essays, documentation, and other dense text and want help at the exact moment confusion appears.

## Problem statement
Readers skip unresolved confusion because context switching to search is costly.

## Promise
**Never read past confusion again.**

## Core workflow
Capture passage -> select exact confusion -> choose explanation mode -> receive explanation modules -> save connected Margin Note.

## Why MARGIN is not a generic chatbot
- It starts from a specific confusion moment in reading
- It targets selected text, not broad conversation
- It stores structured margin notes connected to sources
- It accumulates reusable understanding across reading history

## Version-one scope
- Manual pasted passage input
- Local-first persisted sources, passages, requests, notes, concepts, follow-ups
- Demo explanation generation only

## Deliberately excluded from MVP
- Real AI calls
- OCR/camera ingestion
- Auth/subscriptions
- Cloud sync/backend
- Imported PDF/EPUB readers
- Share extensions
- Knowledge graph visualisation

## Monetisation hypothesis
- Free tier with limited explanations
- Premium tier for unlimited explanations and deeper cross-source concept connections

## Long-term opportunities
- Camera OCR
- iOS share extension / Android selected text processing
- Imported PDFs/EPUBs
- Secure AI explanations
- Cross-source concept graph
- Specialised modules for history, economics, philosophy, and science
